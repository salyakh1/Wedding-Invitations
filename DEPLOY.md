# Инструкции по деплою на Vercel

## Подготовка к деплою

### 1. Настройка Supabase

1. Создайте проект на [supabase.com](https://supabase.com)
2. Выполните SQL скрипт из файла `supabase-schema.sql`
3. Создайте пользователя:
   - Email: `saleh999@example.com`
   - Password: `salean9595$$`

### 2. Подготовка репозитория

1. Создайте репозиторий на GitHub
2. Загрузите код в репозиторий
3. Убедитесь, что файл `.env.local` не попал в репозиторий

### 3. Деплой на Vercel

1. Перейдите на [vercel.com](https://vercel.com)
2. Войдите через GitHub
3. Нажмите "New Project"
4. Выберите ваш репозиторий
5. Настройте переменные окружения:
   - `NEXT_PUBLIC_SUPABASE_URL` - URL вашего Supabase проекта
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Anon key из Supabase
6. Нажмите "Deploy"

### 4. Проверка работы

1. Откройте ваш сайт
2. Войдите с логином `saleh999@example.com` и паролем `salean9595$$`
3. Создайте тестовое приглашение
4. Проверьте публичную ссылку

## Структура URL

- Главная страница: `https://your-domain.vercel.app/`
- Конструктор: `https://your-domain.vercel.app/constructor`
- Публичное приглашение: `https://your-domain.vercel.app/invitation/[id]`

## Возможные проблемы

### Ошибка аутентификации
- Проверьте правильность Supabase URL и ключей
- Убедитесь, что пользователь создан в Supabase

### Ошибки базы данных
- Проверьте выполнение SQL скрипта
- Убедитесь в правильности политик RLS

### Проблемы с загрузкой файлов
- Проверьте настройки Supabase Storage (если используется)
- Убедитесь в правильности CORS настроек
